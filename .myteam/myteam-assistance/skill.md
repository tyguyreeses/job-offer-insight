# Steps
1. Consider the concrete cases the skill or role should enable.
    - What does the user want to accomplish?
    - What multistep processes are involved?
    - What tools or resources are needed?
    - What domain knowledge or best practices should be included?
2. Analyze the current myteam tree.
    - Where does this role or skill fit within the current myteam tree?
    - Should a new role or skill be added?
      - Roles spawn a new agent with only the current context.
      - Skills add instructions to the current agent's context.
3. Summarize your plan to the user.
    - What role or skill will you add?
    - What instructions will you include?
    - How will this help the user accomplish their goal?
4. Implement the role or skill.
    - Follow the plan you summarized to the user.
    - Ensure the instructions are clear, concise, and actionable.

# General Guidelines
- All skills and roles descriptions **must** include a brief descritpion of its purpose, and a very specific set of triggers for when it should be used.
- The instructions within a skill or role should be clear, concise, and actionable. They should provide step-by-step guidance on how to accomplish the task at hand.
- When implementing a new skill or role, consider how it will fit within the existing myteam tree. It should complement and enhance the current structure, rather than creating redundancy or confusion.
- Always summarize your plan to the user before implementing a new skill or role. This ensures that you are aligned with the user's goals and expectations, and allows for any necessary adjustments before implementation.
