## Reference Integration

Use this skill when reference materials are available.

### Optional but recommended

If a project includes a reference folder/resource, suggest loading this skill
before implementing related subsystems.

### Process

1. Identify relevant reference assets
2. Extract patterns worth reusing
3. Document what is reused, adapted, or intentionally not reused
4. Keep final implementation aligned with current project contract

### Rules

1. References inform design; they do not override current requirements
2. Avoid copy-paste architecture without adaptation rationale
3. Record decisions where references influenced structure

### Completion criteria

Reference usage is explicit, justified, and traceable.
