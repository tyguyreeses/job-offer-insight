## Implementation Planning

Create a practical sequence from foundations to feature completion.

### Required plan sections

1. Process setup and branch hygiene
2. Contract/planning docs stage
3. Framework foundations
4. Persistence foundations
5. Test suite planning and build
6. Feature implementation slices
7. Hardening and readiness

### Rules

1. Start with config/bootstrap if the project is config-driven
2. Separate framework prep from feature behavior where possible
3. Include explicit deliverables and exit criteria per phase
4. Keep order dependency-aware and reviewable

### Completion criteria

Plan supports execution without major sequencing ambiguity.
