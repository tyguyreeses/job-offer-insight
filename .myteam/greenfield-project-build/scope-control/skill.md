## Scope Control

Keep the project focused on approved behavior.

### Required practices

1. Mark deferred features explicitly
2. Add placeholders where behavior is intentionally not implemented yet
3. Keep out-of-scope list visible in interface and plan docs
4. Reject unapproved feature expansion during active stage work

### Change control

If scope changes are requested:

1. Update end-goal doc first
2. Update interface contract second
3. Update stage docs and plan third
4. Then implement

### Completion criteria

No hidden scope drift between docs and implementation stages.
